ЗАВДАННЯ №2

- autocomplete атрибут, який дозволяє автоматично заповнити форму, орієнтуючись на початково введені символи, і на те що було ввдеено користувачем в попередніх сесіях. Підтримуєтьтся всіма браузерами(firefox, chrome,safary частково)

- autofocus атрибут, що фокусує курсор в формі, в якій він встановений, при загрузці сторінки. підтримується всіма браузерами крім iOS Safari opera mini

- multiple дозволяє вводити кілька емейлів, наприклад для елементу емейлу або підгрузки кількох файлів підтримується всіма браузерами крім Opera mini, android brows

- placeholder дозволяє вписати текст, який буде видимий для користувача доки форма пуста. всі браузери крім opera mini

- pattern атрибут, який дозволяє ввести певний вираз, і при заповненні користувачем форми, зміст порівнюється з вказаним виразом, і якщо вони не відповідні, то користувачеві виводить вікно, про помилку. всі браузери крім opera mini частково safari, iOS safari

- readonly вказує на те що вказаний в формі текст не редагується а є тільки для ознайомлення

- disabled забарвлює вікно вводу в сірий колір, роблячи його не доступним для користувача. всі браузери, частково IE, Opera mini

ЗАВДАННЯ №3

- range	довзоляє вибрати значення з діапазону відомих, виводиться на екран у вигляді повзунка, яким користувач може водити від одного значення до іншого всі браузери крім opera mini

- tel призначений для вводу користувачем номера телефону, в правильній формі.всі браузери крім opera mini

- time призначений для вводу годин хвилин і секунд, має спеціальні для цього пля і стрілки, яким можна редагувати. не підтримується IE Firefox Sfari, opera mini

- url призначений для вводу url адреси, в заданій формі. всі браузери крім opera mini

- number призначений для вводу числового значення, поле має стрілки для зменшення чи збільшення числа. частково підтримується IE iOS safari


ЗАВДАННЯ №4

- dl, dt, dd ці три теги створюють список визначення (definition) dl створює контейнер списку, dt створює термін, dd визначння цього терміну

- fieldset групує елементи форми, при роботі з великою кількістю даних. Створює рамку навколо полів які входять в його рамки.

- legend використовується для задання заголовка fieldset заголовок буде вказане зверху рамки

- datalist створює список можливих вводів користувачем. стає видими коли пористувач вводить текст або клацне в полі, звязується з текстовим полем за рахунок атрибута id

- optgroup контейнер, який вклюлчає в себе теги <option> і обєднує їх в одну групу під якимось заголовком, заголовок буде виділений жирним і не буде виділитись а всі елементи списку будуть зсунуті чуть вправо від заголовка

- progress тег для демонстації прогресу якогось процесу показується у вигліді стрічки, яка заповнюється у процесі прогресу

- meter виводить графічно значення в якомусь відомому діапазоні, наприклад можна задати значення температури якогось обєкту і даний тег буде графічно в кольорі показувати критично низьку, і нормальту температуру

- iframe тег, що дозволяє помістити в документ, будь який інший докумен, або цілий сайт, можна коригувати розмір фрейму

- embed, object існують для відображення і загрузки роликів, відео, якщо браузер не підтримує якісь відео, то можна вкладати за допомогою іншого такого ж тега, якийсь графічний малюнок,
